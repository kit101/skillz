# 会话回放输出规则

## 基本格式规范

### 系统初始化消息
```
╭─────────────────────────────────────────────────────────────────────────────╮
  # System Init                                                                
  cwd:    /workspace                                                           
  tools:  "available","tools","list"                                           
  session_id: unique-session-identifier                                        
  mcp_servers:                                                                 
          server-name: status                                                  
  model:  model-name                                                           
  permission_mode: default                                                     
╰─────────────────────────────────────────────────────────────────────────────╯
```

### 用户请求消息
```
> 用户的具体请求内容
```

### 助手思考和规划
```
✦ 助手的思考过程和行动计划说明
```

### 工具调用执行
```
╭─────────────────────────────────────────────────────────────────────────────╮
  ✓  ToolName (Tool Description) {"parameter": "value"}                       
                                                                               
     工具执行的详细输出内容                                                    
     可能包含多行结果信息                                                      
╰─────────────────────────────────────────────────────────────────────────────╯
```

### 最终结果总结
```
✦ 完成！任务执行结果的总结说明

   结果摘要：
    - 关键指标1：值
    - 关键指标2：值
    - 状态：成功/失败
    
   后续建议或注意事项...
```

## 格式化规则

### 视觉元素
- 使用 `╭─╮╰─╯` 字符绘制边框
- 使用 `✦` 标识助手的重要说明
- 使用 `✓` 标识成功的操作
- 使用 `>` 标识用户输入
- 适当使用空白行分隔不同部分

### 信息层次
1. **系统信息**: 框线包围，顶部标识
2. **用户请求**: 简洁的 `>` 前缀
3. **助手思考**: `✦` 引导的自然语言说明
4. **工具执行**: 详细框线展示，包含输入参数和输出结果
5. **最终总结**: `✦` 开头的综合结论

### 内容组织原则
- **时间顺序**: 严格按照会话发生的时间顺序排列
- **因果关系**: 清晰展示工具调用与结果之间的逻辑关系
- **关键信息突出**: 重要数据和状态变化要明显标示
- **冗余去除**: 避免重复展示相同信息

### 最后统计

## 特殊情况处理

### 错误处理
```
╭─────────────────────────────────────────────────────────────────────────────╮
  ❌  ToolName (Error Description) {"parameter": "value"}                     
                                                                               
     错误详情和可能的解决方案                                                  
     错误代码: ERROR_CODE                                                      
     建议操作: 具体的修复建议                                                  
╰─────────────────────────────────────────────────────────────────────────────╯
```

### 多轮交互
对于复杂的多轮工具调用，保持每轮的完整记录，使用适当的分隔符区分不同的交互轮次。

### 长内容处理
对于特别长的输出内容，可以：
1. 显示关键摘要信息
2. 提供完整内容的截断版本
3. 在必要时提供查看完整内容的方法
